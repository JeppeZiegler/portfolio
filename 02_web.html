Tema 02_Web

Temabeskrivelse

I tema 2 blev vi introduceret for nogle af de grundlæggende redskaber for en multimediedesigner. Ved hjælp af VS code lærte vi mere om opsætningen af websites i html og css. Og om behandlingen af egne billeder og hertil også hvordan man implementerer ovennævnte i kode. Vi arbejdede også med responsivt design ud fra tilgangen "mobile first", hvor vi lærte om css grids og mediaqueries. Herudover blev vi også præsenteret for en række faglige begreber inden for design af digitale brugergrænseflader, vi snakkede om gestaltlove, designkonventioner, visuelt heraki, kontraster og farvers grundprincipper. Til sidst i forløbet lavede vi vores studiestartsprøve, ud fra uddelte layoutdiagrammer og wireframes.

  • Viden om en multimediedesigners rolle i erhvervets praksis
  • Viden om forskellige redskaber til billedebehandling
  • At finde og anvende rettighedsfrie billeder

Gennemgang af temaet

Mere fra VS code

Under forløb 02_web lærte vi mere om programmet VS kode. Vi lavede en par websites med forskelligt indhold, herunder en kæledyrs hjemmeside, og en med vores familie (visitkort). Siderne var lavet med vores egne billeder, det medfører at vi også snakkede om omplementering af dem og rettigheder på websites. Målet var at blive mere genkendelig med programmet og værktøjerne i VS code, som er en af de grundlæggende redskaber for en multimediedesigner og som vi i øvrigt også skulle bruge i slutningen af forløbet til vores studiestartsprøven.

I undervisningen fik vi gennemgået nogle centrale begreber omkring design af brugergrænsefladen, hvor vi i øvelsen 20 bokse, kunne prøve kræfter af med css, hvor man lærte nogle grundlæggende properties til at får at få boksene til at ligne dem øverst.

Opsætning af html

I denne opgave skulle vi lave vores første rigtige website, som var studiestartsprøven. Siderne skulle være responsive, der både fungerer i web version og i mobil version. Og vi lærte om tanken "mobile first", som er at man opsætter sin html kode, som det skal se ud i mindre skærmstørrelse. Til opgaven fik vi udleveret billeder og lorem ipsum tekst som vores indhold. Tilgengæld skulle vi selv finde fonte, på Fontjoy, samt style dem i CSS og implementere dem i koden. Udover indhold fik vi udleveret layoutdiagram og wireframes som sitet er bygget op omkring.

02.01.02 Studiestartsprøven

Studiestartsprøven vi lavede var ud fra helt basal HTML kode. Vi lærte om grundlæggende semantisk opmærkning og de forskellige tags man kan bruge og hvad de kan. Vi snakkede om, head, body, header, main, footer og tekst (h1,h2,p) Derudover lærte vi at sætte klasser og id på HTML tags. Klasser kan sættes på flere tags, hvorimod et id kun kan bruges én gang. Ved hjælp af klasser og id, kunne vi style vores website i CSS, med farver, fonte og grid. Grid skulle vi bruge til at opsætte vores kode ud fra det udleverede layout. Det gør man grundlæggende ved at "display: grid" på dine klasser, og så styre hvor man grid-template-columns eller grid-template-rows man vil have, så sætter css koden op efter designet. Så hvis man eksempelvis sagde (display:grid; grid-template-columns: 1fr 1fr;) så ville man få tingene sat op ved siden af hinanden. Med det i tankerne blev vi også introduceret til mediaqueries, som gør at man kan bestemme hvornår en css property skal gælde. Hvis man eksempelvis satte et grid op under (@media screen and (min-width: 800px)), så ville det kun sættes i grid når skærmstørrelsen er over 800px.
